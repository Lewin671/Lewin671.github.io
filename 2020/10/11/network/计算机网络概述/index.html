计算机网络概述

Posted on 2020-10-11 Edited on 2020-10-12 In 计算机网络

计算机网络是指将有独立功能的多台计算机，通过通信设备线路连接起来，在网络软件的支持下，实现彼此之间资源共享和数据通信的整个系统。它是计算机技术和通信技术相结合的产物。

计算机网络的基本功能是数据通信和资源共享。

计算机网络的组成和拓扑结构

计算机网络的组成

计算机网络从结构上可以分为两部分:

  • 负责数据处理的计算机和终端
  • 负责数据通信的通信控制处理机和通信线路。

从逻辑上可以分为两个子网:

  • 资源子网。处理数据业务，向网络用户提供网络资源和网络服务。如网络终端、外部设备（打印机等）、各种软件资源与数据资源等。
  • 通信子网。 主要完成数据传输、转发等通信处理工作。

计算机网络的拓扑结构

所谓网络的拓扑结构是指网络中各个节点间的互连模式，也就是网络链路与节点的几何布局，它定义了各个节点间的物理和逻辑位置。常见的网络拓扑有5种：网状、星状、树形、总线型和环状。

网状拓扑

  1. 避免拥塞问题
  2. 具有较好的健壮性
  3. 具有较好的安全性。（只有预期的接受者才能收到信息）
  4. 便于管理
  5. 安装费用高

星状拓扑

  1. 拓扑结构简单
  2. 具有较好的健壮性
  3. 便于管理
  4. 中央控制器是网络的瓶颈

树状拓扑

  1. 拓展了网络距离
  2. 允许网络隔离不同的计算机的通信

总线型拓扑

  1. 不存在路由
  2. 易安装
  3. 总线长度有限
  4. 故障隔离问题

环形拓扑

  1. 易安装
  2. 当环中节点过多时，势必影响信息传输速率，使网络的响应时间延长
  3. 环路是封闭的，不便于扩充
  4. 可靠性低，一个节点故障，将会造成全网瘫痪

计算机网络体系结构

计算机网络的各个层次及其协议的集合称为体系结构。

网络协议和分层

要进行正确的传输，每个结点就必须遵守事先约定好的规则，这些规则就是网络协议。

网络协议主要有以下3个要素:

  • 语法: 即数据与控制信息的结构或格式，如数据格式、信号电平等规定。
  • 语义: 即需要发出何种控制信息，完成何种动作，以及做出何种应答，包括用于调整和进行差错处理的控制信息。
  • 时序（同步）: 即事件实现顺序的详细说明，包括速度匹配和顺序。

对于复杂的网络协议，其结构最好是采用层次式的。

ISO OSI参考模型

开放式系统互联模型（英语：Open System Interconnection Model，缩写：OSI；简称为OSI模型）是一种概念模型，由国际标准化组织（International Organization for Standarlization, ISO）提出，一个试图使各种计算机在世界范围内互连为网络的标准框架。

OSI中采用了上图所示的七层体系结构。

  1. 物理层

    物理层规定在一个结点内如何把计算机连接到通信介质上。在物理层中，数据的传送单位是比特(bit)。

  2. 数据链路层

    数据链路层把两个相邻结点不可靠的物理链路变成可靠、无差错的逻辑电路，具有把原始比特流进行分帧、排序、设置检错、确认、重发、流量控制等功能。数据链路层的传输信息的单位是帧(frame).

  3. 网络层

    网络层连接网络中任何两个计算机节点，从一个节点上接受数据并正确地传送到另一个节点。 在网络层传送的信息单位是分组或包(packet)。网络层的主要任务是选择合适的路由和交换节点。

上述三层组成了所谓的通信子网，用户计算机连接到通信子网上。 通信子网负责把一个计算机上的数据可靠地传送到另一台计算机，但是并未实现两台主机上的通信。

  1. 传输层

    传输层真正实现了“端到端”通信，把数据可靠地从一方的用户进程或程序发送到另一方的用户进程或程序。 这一层的控制通常由通信两端的计算机完成，中间节点一般不提供该层服务。 传输层传输的单位是报文(message)。

  2. 会话层。

    会话层允许两台计算机上的用户进程建立会话连接，双方相互确认身份，协商对话连接的细节。 会话层还提供同步机制，在数据流中插入同步点机制，在每次网络出现故障后可以仅重传最近一个同步点以后的数据，而不必从头开始。

  3. 表示层。

    表示层主要解决用户信息的语法表示问题。表示层将数据从适合于某一系统的语法转变为适合于OSI系统内部使用的语法。数据加密、解密、信息压缩等等都是本层的典型功能。

  4. 应用层。

    应用层确定进程之间通信的性质，以满足用户的需要。它负责用户信息的语义表示，并在两个通信者之间进行语义匹配。这一层包括最终用户普遍需要的协议，如虚拟终端协议，文件传输协议、电子邮件协议等等。

层间通信和对等通信

在计算机网络中，每层的功能是由该层的实体来实现的。实体之间需要通过通信交换数据，下层实体需要向上提供服务，上层实体需要调用下层实体所提供的服务完成自己的功能。

网络中每层使用定义好的协议与其相邻层通信。上图用OSI分层表示两台网络主机之间的连接，每层之间的箭头表示它们之间的通信通道。

同一个网络结点上相邻层次中实体之间的通信称为层间通信。从概念上讲，两台主机进行通信时，它们相应层也进行通信。我们把不同主机内相对应的层次称为对等层，对等层之间两个通信实体称为对等层实体。 对等层之间的通信叫做虚通信。 层间通信以及物理层之间的通信叫做实通信。

数据传输的方式

数据在网络中各节点是沿着层次传送的。如上面的图所示，数据从发送进程出发，在应用层加上报头AH后，送到表示层。表示层不能区分AH和后面的数据部分，而是把这两部分看成一个整体，作为应用层来的数据，简单地加上表示层的报头PH后送到会话层。重复此过程，一直传送到物理层。 数据链路层与其他层不同的是，在数据链路层不仅要加上报头DH，还要加上报尾DT。

服务、服务访问点和数据单元

在ISO OSI/RM模型中，设物理层为第1层，应用层为第7层，在N层和N+1层的接口处，由N层向N+1层提供服务。 这里N层是服务提供者，而N+1层则是该服务的用户。

层间服务处提供服务的地方称为服务访问点（Service Access Point, SAP），每个服务访问点都有一个唯一的地址标识符。

相邻层在提供服务的过程中要传递信息，这些信息单位在OSI模型中称为服务数据单元(Service Data Unit, SDU)。 在N+1和N层间传递的数据单元，记为SDU。 N层的PDU由该层的SDU加上该层的协议控制信息(Protocal Control Information, PCI)构成。

面向连接和无连接的服务

在通信的角度来看，OSI参考模型中，下层能向上层提供两种不同形式的服务: 面向连接的服务和面向无连接的服务。

  1. 面向连接的服务。

    所谓连接，就是两个对等实体为进行数据通信而进行的一种结合。面向连接的服务在进行数据交换前先建立连接，当数据传输结束后应释放这个连接。

    这种连接可以避免报文的丢失、重复和乱序。

  2. 在无连接服务的情况下，两个实体之间的通信不必事先建立一个连接。相对于面向连接的服务，无连接服务灵活方便且快速，但是它不能防止报文的丢失、重复和乱序。

    面向无连接服务主要有三种类型:

    • 数据报。它的特点是发完报文就结束。

    • 证实交付。又称可靠的数据报，这种服务对于每一个报文产生一个证实给发送方，不过这种证实不是来自对应方用户，而是来自提供服务的层。

      这种证实只能保证报文已经发给目的站，但是不能保证对应方用户正确地收到报文。

    • 请求回答。 如果接收端发现报文有错误，则回送一个表示有错误的报文。但是双方发送的报文都有可能丢失。
