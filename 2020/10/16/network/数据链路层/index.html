数据链路层

Posted on 2020-10-16 Edited on 2020-10-19 In 计算机网络

数据链路层的任务是提供两个相邻结点或主机及其相连的网络结点之间可靠通信。 数据链路层提供的主要服务是线路规程、差错控制和流量控制。

线路规程

通信至少需要两个一起工作的设备，一个接收、一个发送。 这就需要双方进行一些协调工作。线路规程解决了“现在该谁发送”的问题。

数据链路层的线路规程监视链路的建立，并在给定时刻分配一个具体设备进行数据传输。线路规程可以两种模式实现: 询问/应答模式和轮询/选择模式。

询问/应答模式

当两个设备之间存在一条专用链路时，即点对点方式，采用询问/应答确认模式。

询问/应答模式负责协调哪个设备可以开始传输以及预期的接收方是否已经就绪并可以进行接收。采用ENQ/ACK模式，只要一条链路两头的设备级别相同，任意一个设备都可以启动一个会话过程。

询问/应答模式的工作方式为: 启动方首先发送一个询问帧(ENQ)，询问接收方是否可以就收数据。接收方如果已经准备好接收，必须回答一个应答帧(ACK)。 如果没有准备好，则必须回答一个否定应答帧(NAK)。发送方发送一个询问帧之后，会有三种可能的情况:

  • 没有回应. 如果在一定的时间期限内既没有接收到应答帧也没有接收到否定帧，启动方就假定询问帧在传输中丢失了，断开连接并重新发送一个副本。 一个启动系统通常在放弃努力之前为建立链路进行三次这样的尝试。

  • 回应是否定的。 如果对ENQ帧的回应是否定的，并尝试三次都失败了，启动方就断开连接，并在另一个时刻重新开始整个过程。

  • 回应是肯定的。如果回应是肯定的，启动方就可以随时发送数据了。一旦将所有数据都发送完毕，发送系统以一个传输结束帧(EOT)结束传输。

轮询/选择模式

轮询/选择模式是将一个设备设计为主设备，而其他设备作为从设备的拓扑结构中采用的。多点连接系统必在多个站点，而不仅是两个站点之间进行协调。因此，在这类情形下，问题不仅仅是确定设备是否就绪，还要确定哪一个站点有权使用信道。

  1. 工作方式

    一个多点连接由使用同一条传输线路的一个主设备和若干从设备组成，所有的信息交换都必须通过主设备进行。主设备负责控制链路，从设备遵从其指令。必须由主设备来决定在给定时刻哪一个设备有权使用信道，因此主设备是会话的发起者。如果主设备希望接收数据，它告知目标从设备准备好接收数据，这个功能称为选择。

  2. 地址问题

    轮询/选择协议对每一帧都要一个地址进行标识，指明它是从一个设备而来，或到一个设备去。在任何一次传输中，这个地址会在每一帧的特定部分出现，这一部分叫做地址域。如果该帧是从主设备而来，地址指明了数据的接收者；如果该帧是从从设备而来，地址指明了数据的产生者。

  3. 选择

    每当主设备有数据发送时就采用选择模式。主设备虽然控制着链路，但是它不知道接收数据的设备是否准备好接收数据。因此，主设备必须告知从设备将有数据到来，然后等待从设备处于接收就绪的应答信号。

    在发送数据之前，主设备产生并发送一个选择帧(SSL)，并在帧的地址域中填入接收数据的从设备地址。 多点拓扑在多个设备间共用一条链路，所以链路上的任何一帧都可以被链路上的任何一个设备接收。 当帧沿着链路传输时，每个设备都检验其中的地址域。 只有当一个设备发现该地址域是自身时才打开该帧并读取数据。

  4. 轮询

    轮询功能是主设备请求从设备进行传输。在没有接收到请求前，从设备是不允许发送数据的。所有的控制都集中在主设备。 当主设备准备好接收数据时，它必须依次询问从设备是否有数据发送，被询问到的设备如果有数据发送则发送数据帧，如果没有数据发送则以否定应答帧作为回答。

流量控制与差错控制

流量控制有两个要点:

  • 数据流不能使接收方过载。
  • 接收方对数据进行确认应答。

差错控制主要指错误检测和重传方法。一个帧中出现任何一个错误，接收方就返回一个否定应答帧，出错的帧就被发送方重新传送，这个过程称为自动重传。

数据被重传有三种情况: 帧破坏、帧丢失和应答帧丢失。

流量控制和差错控制是结合在一起实现的，主要有两种实现协议： 停止等待协议和滑动窗口协议。

停止等待协议

流量控制

在停止等待协议(Stop and Wait)中，发送方每发送一帧后就等待一个应答帧。只有当接收到应答帧之后，发送方才接受下一帧。 这种发送和等待交替的过程不断重复，直到发送方发送一个传输结束符。

优点: 协议简单

缺点: 效率低

从上图可知，在无差错的情况下，完成一帧的发送至少需要如下一个窗口时间WTW_{T}WT​，其中

WT=tI+2tP+2tProc+tSW_{T} = t_{I} + 2t_{P} + 2t_{Proc} + t_{S} WT​=tI​+2tP​+2tProc​+tS​

其中tIt_{I}tI​是发送一个数据帧的时间（发送时延），tPt_{P}tP​是电信号在物理链路上的传输延迟，tSt_{S}tS​是应答帧的发送时间，tProct_{Proc}tProc​是收到数据帧后的处理时间。

差错控制

为了实现差错控制，停止等待协议采用ARQ(Automatic Repeat reQuest)技术，ARQ技术处理错误的三种情况:

  • 帧破坏（校验码）
  • 数据帧丢失（超时）
  • 应答帧丢失（超时）

停止等待协议中的差错控制的要求如下:

  1. 发送设备要保留数据帧的备份，直到数据帧被正确接收位置
  2. 数据帧和应答帧都必须交替的标识为0和1。一个标号为1的数据帧被一个标号为0的应答帧所确认，说明当前接收方已经接收到了数据帧1，期望接收数据帧0. 如果接收方收到了两个相邻的数据帧，且标号相同，说明接收方收到了一个重复帧，应当丢弃一个重复帧。
  3. 如果在数据帧中发现一个错误，说明该帧在传输中发生了错误，就会返回一个否定应答帧。 否定应答帧是不编号的，它通知发送方重新发送最近的一帧。
  4. 发送设备安装一个定时器。如果在规定时间内不能收到预期的应答信息，发送方就假定最近的一帧数据已经在传输过程中丢失并再次发送它。

注: 由于数据链路层一般不会出现ACK迟到的情况，所以不需要给ACK进行编码。

同时，定时器设置的超时时间应仔细选择，一般可设置为略大于“从发送方到接收方的平均往返时间”。

停止等待协议的效率分析

这里的分析忽略了对数据分组的处理实验以及对确认分组的处理时延。由于TAT_{A}TA​一般远小于TDT_{D}TD​，因此信道利用率可以这样估计:

U=TDTD+RTTU = \dfrac{T_{D}}{T_{D} + RTT} U=TD​+RTTTD​​

当RTTRTTRTT远大于TDT_{D}TD​时，信道占用率将会变得非常低。

信道利用率是信道被占用时间与总时间之比。

滑动窗口协议

流量控制

在流量控制的滑动窗口协议中，发送方在收到应答消息前可以发送若干帧。帧可以直接依次发送，这意味着链路上可能同时存在几个数据帧，从而充分利用了链路的传输能力。

在发送方，只要窗口未被填满就可以在未收到应答帧的情况下继续发送数据帧。发送窗口用于存放已发送但未收到应答的数据帧和在收到应答之前可以发送的数据帧。而接收窗口用于存放已接收但未给应答的数据帧。

在滑动窗口协议中，数据帧的编号以模nnn方式编号。也就是说，编号从000到n−1n-1n−1。窗口的大小是n−1n-1n−1。

注意： 窗口的大小不是nnn.

接收方发送的应答帧编号是接收方希望接收到的下一帧的编号。

这种确认方式叫做累积确认。

例如，接收方发送了一个编号为5的应答帧，表示接收方接收到了5号以前的数据帧，希望接收方发送5号帧。

发送窗口移动的例子

发送窗口分为两部分，左边部分已经发送但是还没有收到信息的数据帧，右边部分还未发送但是可以发送的数据帧。 每发送一帧，左右两部分的分界就向右移一帧，每收到一个应答帧，整个发送窗口就右移若干帧. 移动的距离是最后一次银达镇的编号和现在接收到的应答帧的编号的差值（模n）.

当收到了编号为3的应答帧之后，表示接收方已经收到了3号之前的所有数据帧，因此窗口往右移3格。（图c)

接收窗口移动的例子

接受窗口也分成两个部分，左边部分是已经接受但是还没有发送应答信息的数据帧，右边部分是可以被接受的数据帧。 只有编号处于窗口内的帧才是可以被接受的帧。每接收一帧，左右两部分的分界就往右移一帧；每发送一个应答帧，整个接收窗口就向右移若干帧。移动的距离是最后一次应答帧的编号和现在发送的应答帧的编号的差值模nnn。

差错控制

下面开始解决滑动窗口协议的差错控制，有两种技术: 回退n自动重复请求和选择拒绝自动重复请求。

为了解决帧丢失和损坏帧的重发问题，需要在基本的滑动窗口流量控制上增加3个特性:

  1. 在发送出去的帧被应答信息确认前保持它们所有的备份。这就是前面所说的，帧被发送后未被确认前仍留在滑动窗口中的原因。
  2. 除了应答帧外，接收方还可以发送否定应答帧以通知发送方重新发送一个损坏的帧。因为滑动窗口是一个连续传输的机制，应答帧和否定帧都必须带有编号以便识别；应答帧带有期望收到的数据帧的编号，而否定帧带有损坏帧的编号，都是接收方期望接收的数据帧的编号。
  3. 和停止等待协议自动重复请求ARQ类似，在滑动窗口自动重复请求协议中，发送设备具有一个定时器来使得发送方可以处理帧丢失的情况。

回退n自动重复请求

在滑动窗口回退nnn自动重复请求中，如果一帧丢失或者损坏，则该帧以及后面的帧都必须进行重传。 差错情况一般有三类: 帧破坏、帧丢失和应答帧丢失。

  1. 帧破坏。接收方发现数据帧被破坏，于是向发送方发送NAK应答帧，告诉发送方该帧被破坏。此时发送方拒绝接受后面的帧，直到正确接收到该编号的数据帧。 而发送方将会从被破坏的帧开始，对后续窗口内的数据帧都进行重传。
  2. 数据帧丢失。如果某个数据帧丢失了，接收方的定时器如果超时，则会发送该数据帧的应答帧NAK，告诉发送方没有接收到该数据帧。于是发送方会从该超时的帧开始，重传后续窗口内的所有数据帧。
  3. 应答帧丢失。应答帧丢失后，发送方的定时器将会超时，从而操作对该据数据帧进行重发。由于接收方已经接收到了该数据帧，于是将删除重复的数据帧，并重新发送应答帧。
  4. 窗口的大小与编号范围的关系。在回退nnn协议中，如果帧的编号范围是0∼n−10 \sim n-10∼n−1，则窗口的尺寸为n−1n-1n−1。

选择拒绝自动重复请求

在选择拒绝自动重复请求中，只有特定丢失或损坏帧被重发。接收方接收到的数据帧可以是不按顺序到达的。选择拒绝自动重复请求和回退nnn自动重复请求有以下的不同:

  1. 接收设备必须具有排序的逻辑功能。同时，接收方在发送了NAK帧之后，必须存储所收到的所有帧。直到损坏的帧被重新收到为止。
  2. 发送设备必须具有查找机制，以便发现和选择需要重传的帧。
  3. 在所有重传帧被排序和所有重复帧被辨别出来并删除之前，所收到的所有帧都必须保存。
  4. 为了有助于选择，ACK帧的编号是指被正确接收的帧，不是指期望接收的帧号。

下面分析选择拒绝自动重复请求协议中如何处理帧破坏的、帧丢失和应答帧丢失的。

  1. 帧破坏。下图显示了一个帧被破坏的情况。接收方返回了一个对数据帧2的否定应答帧NAK2，说明数据0和数据帧1都被正确接收了，而数据帧2有误。发送方只重传数据帧2，而不需要重发已经发送的数据帧3、4、5。接收方继续接收数据帧3、4、5而不是丢弃它们，但是不能对数据帧3、4、5作应答，因为数据帧2还未被正确接收。当重发的数据帧2到达接收方后，接收方对接收的数据帧进行排序，返回一个ACK5，作为对数据帧2，3，4，5的应答。

  1. 帧丢失。下图显示了数据帧丢失的情况。尽管数据帧可以不按顺序接收，但是不能不按照顺序应答。数据帧2在传输中丢失了，接收方接收到数据帧3时，会发现数据帧2丢失，向发送方发送NAK2. 如果丢失的是最后一帧，则接收方不作任何相应，发送方当做应答帧丢失一样处理。

  1. 应答帧丢失。 担当发送窗口满或者传输完毕时，启动定时时钟。 如果在预定时间段内没有应答帧到来，发送方将尚未应答的所有帧都重传一遍。这和回退nnn协议是一样的。

窗口的大小与编号范围的关系。如果帧的编号是000到n−1n-1n−1，则发送窗口尺寸和接受窗口尺寸之和应小于或等于nnn。如果要求发送窗口和接受窗口大小相等，则窗口尺寸应该小于等于n2\dfrac{n}{2}2n​.

滑动窗口协议的效率

当nTD≤TD+RTTnT_{D} \le T_{D} + RTTnTD​≤TD​+RTT时，信道利用率

U=nTDTD+RTTU = \dfrac{nT_{D}}{T_{D} + RTT} U=TD​+RTTnTD​​

当nTD>TD+RTTnT_{D} > T_{D} + RTTnTD​>TD​+RTT时，信道利用率接近100%100\%100%.因此，窗口应该选择得足够大，以保证在第一个应答帧返回到发送方之前，发送方可以继续发送，即

nTD>TD+RTTnT_{D} > T_{D} + RTT nTD​>TD​+RTT
