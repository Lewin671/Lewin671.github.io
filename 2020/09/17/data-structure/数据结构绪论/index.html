数据结构绪论

Posted on 2020-09-17 In 数据结构

数据结构的概念

数据： 是指对象的表示。即按照适用于通信、解释或处理的方式所形成的关于事实、概念或者指令的表示。

数据只是表示，并无具体含义。

数据元素:\quad 数据元素是数据的基本单位。

数据对象： \quad具有相同性质的数据元素的集合，是数据的一个子集。

数据结构：相互之间存在一种或多种特定关系的数据元素的集合。

一般认为，一个数据结构是由数据元素依据某种逻辑联系组织起来的。对数据元素间逻辑关系的描述称为数据的逻辑结构。

而数据必须在计算机内存储，数据的存储结构是数据结构的实现形式，是其在计算机内的表示。

有些学者认为，数据结构应该由数据的逻辑结构、数据的存储结构及其运算三部分组成。

数据的逻辑结构

数据的逻辑结构可分为两大类。

  • 线性结构。其特点是，若线性结构的结点数为1，则该结点既是始结点又是终结点； 若线性结构的结点数大于等于2，则有且仅有一个始结点和一个终结点，始结点有一个后继节点，中间节点有一个前驱结点和一个后继节点，终结点有一个前驱结点。
  • 非线性结构： 结构中的结点可能又多个前驱和多个后继结点。非线性结构主要包括集合、树、图。

数据的存储结构

数据的存储结构（或称物理结构）是指逻辑结构在计算机中所需的存储空间、空间的构成结构以及对存储结构的访问方式等的总称。

一般地，数据的存储结构有顺序、链接、索引和散列。

顺序存储

在计算机中用一组地址连续的存储单元依次存储线性表的各个数据元素,称作线性表的顺序存储结构。

链式存储

在计算机中用一组任意的存储单元存储线性表的数据元素(这组存储单元可以是连续的,也可以是不连续的)。它不要求逻辑上相邻的元素在物理位置上也相邻。因此它没有顺序存储结构所具有的弱点,但也同时失去了顺序表可随机存取的优点。

索引存储

除建立存储结点信息外，还建立附加的索引表来标识结点的地址。索引表由若干索引项组成。其特点有:

索引存储结构是用结点的索引号来确定结点存储地址，其优点是检索速度快，缺点是增加了附加的索引表,会占用较多的存储空间。在数据表中，就是用索引键来进行存储与检索的。

散列存储

散列存储，又称hash存储，是一种力图将数据元素的存储位置与关键码之间建立确定对应关系的查找技术。散列法存储的基本思想是：**由节点的关键码值决定节点的存储地址。**散列技术除了可以用于查找外，还可以用于存储。

通过hash function来计算索引值，索引表求出结点的指针地址，是索引方法的一种延伸和拓展。

对数据结构的操作

几种数据逻辑结构上的常用操作有:

  1. 插入
  2. 删除
  3. 修改
  4. 排序
  5. 查找

算法

一个算法就是给出求解特定类型问题的运算序列的一个有穷规则集合。

算法的特征

算法的5个重要特性:

  1. 有限性(有穷性): 一个算法必须总是（对任何合法的输入值）在执行有穷步之后结束，且每一步都可在有穷时间内完成。
  2. 确定性：算法中每一条指令必须有确切的含义，读者理解时不会产生二义性。即对于相同的输入只能得出相同的输出。
  3. 输入： 一个算法有0个或多个输入。
  4. 输出：算法有1个或多个输出。
  5. 可行性：能够实现，算法中描述的操作都是可以通过已经实现的基本运算执行有限次来实现。

算法的评价准则

评价一个算法的优劣，通常考虑以下5个方面:

  1. 正确性。
  2. 时间复杂性。
  3. 占用空间。
  4. 可读性。
  5. 鲁棒性(robustness)。对有缺失、有噪声或有错误的输入数据，算法应该具有较强的适应能力。

算法分析基础

算法分析一般分为两个阶段： 事前分析和事后测试。
